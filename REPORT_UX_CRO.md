# REPORT_UX_CRO

## Проблемы, мешавшие доверию и покупке
- Резковатые межстраничные переходы снижали ощущение надежности.
- Служебная CTA "Продолжить активацию" ранее могла вести себя нестабильно по размеру.
- Неоднородные состояния интерактивных элементов ухудшали предсказуемость UI.
- Технические страницы (cart/payment/redeem/status) рисковали смешиваться с SEO-трафиком и шумить в выдаче.

## Что улучшено

### 1) Плавность и предсказуемость навигации
Файлы:
- `assets/js/app.js`
- `assets/js/app.min.js`
- `assets/css/unified-premium.css`

Изменения:
- Переходы применяются только для реальной внутренней навигации (intent-flag).
- Синхронизированы тайминги leave/enter и смягчены easing-кривые.
- Добавлена очистка transition-классов на `pageshow`.

Эффект для UX:
- Убрано ощущение "дерганного" интерфейса.
- Визуально повышается качество и “production feel”.

### 2) Стабильность CTA
Файл:
- `assets/css/unified-premium.css`

Изменения:
- Финальный safety override для `.gptishka-resume-activation`: ограничение ширины и fit-content.

Эффект для CRO:
- CTA не разваливает layout и не теряет клик-приоритет.

### 3) Снижение визуального/поведенческого шума
Файлы:
- `assets/js/app.js`
- `assets/js/app.min.js`
- `assets/js/support-widget.js`

Изменения:
- Отложенная гидратация pulse-эффекта кнопок.
- Ограничение тяжелых эффектов только релевантным страницам/устройствам.

Эффект:
- Меньше микролагов на слабых устройствах.
- Стабильнее восприятие ключевых блоков тарифа/оплаты.

## Что усиливает конверсию уже сейчас
- Более “чистое” ощущение пути: `Тариф -> Модалка -> Оплата`.
- Уменьшение фризов/рывков на интерактиве CTA.
- Более предсказуемые состояния кнопок и переходов.

## Что протестировать A/B в следующей итерации
1. Текст и контраст primary CTA в hero (вариант с более коротким action copy).
2. Sticky мини-summary корзины на мобильных в checkout-сценарии.
3. Порядок блоков доверия над/под тарифной сеткой.
4. Отдельные варианты microcopy для promo code ошибок (короткий vs. подробный).
5. Триггер появления support-widget (по таймеру vs. по скроллу).
